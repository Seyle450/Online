Analytics Einbindungs-Snippet

Kopiere den folgenden Code und füge ihn in jede HTML-Seite ein, die du tracken möchtest.

Einbindung in den <head> (empfohlen)

<!-- Analytics Tracker – vor dem schließenden </head> einfügen -->
<script src="/analytics/tracker.js" defer></script>
Wichtig: Stelle sicher, dass /analytics/tracker.js auf GitHub Pages erreichbar ist. Wenn deine Website in einem Unterordner liegt (z.B. /portfolio/), passe den Pfad entsprechend an.

Alternative: Einbindung vor </body>

<!-- Vor dem schließenden </body> einfügen -->
<script src="/analytics/tracker.js"></script>

Komplettes Beispiel einer HTML-Seite

<!DOCTYPE html>
<html lang="de">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Meine Seite</title>

  <!-- ↓↓↓ Einzige Zeile die du hinzufügen musst ↓↓↓ -->
  <script src="/analytics/tracker.js" defer></script>
  <!-- ↑↑↑ Fertig! ↑↑↑ -->

</head>
<body>
  <!-- Dein Seiteninhalt -->
</body>
</html>

Tracker konfigurieren (tracker.js anpassen)

// In /analytics/tracker.js – einmalig anpassen:

// TODO: Diese Zeile mit deiner echten Worker-URL ersetzen
var WORKER_URL = 'https://TODO-DEIN-SUBDOMAIN.workers.dev';
//                  ↑ nach `wrangler deploy` ausgegeben

Seiten die du tracken solltest

/* Füge den Script-Tag in alle diese Dateien ein: */

index.html                    ← Portfolio-Hauptseite
kunde-a/index.html            ← Kundenwebsite A
kunde-b/index.html            ← Kundenwebsite B
// ... weitere Seiten nach demselben Muster
Das Tracking-Script lädt asynchron und blockiert nie die Seite. Falls der Worker nicht erreichbar ist, passiert einfach nichts – kein Fehler, kein Absturz.
